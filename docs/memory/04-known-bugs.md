# Известные баги

_Обновлено: 2026-04-25_

## Открытые

### Truncation транскрипта на ~3500 символов
- **Где**: bot.py, отправка финального транскрипта в Telegram
- **Симптом**: длинные транскрипты обрезаются в одном сообщении
- **План фикса**: split на multiple 4096-char Telegram messages
- **Приоритет**: средний

### VK timeout >10 мин
- **Где**: download-цепочка для VK
- **Симптом**: видео >10 мин не скачиваются с VK
- **План фикса**: причина не диагностирована, гипотезы (HLS, размер чанков) не проверены
- **Приоритет**: низкий (мало пользователей VK)

### Cut-кнопка повторно спрашивает URL
- **Где**: ConversationHandler в bot.py, флоу после нарезки
- **Симптом**: после получения cut-видео, кнопка "сократить" просит ссылку заново вместо того чтобы использовать сохранённую
- **План фикса**: проверить какой state теряет URL — нужно искать в ConversationHandler
- **Приоритет**: UX-косметика, средний

### Copyright-рефьюз форматтера
- **Где**: format_transcript() в transkrib-api
- **Симптом**: на текстах песен Claude отказывается форматировать со ссылкой на copyright
- **План фикса**: detect-and-fallback — ловить отказ → отдавать сырой транскрипт без форматирования. Промт-обход не сработает (это hardcoded в Claude).
- **Приоритет**: низкий (проявляется только на песнях/lyrics)

## Не блокирующие warnings

- **PTBUserWarning per_message=False** — bot.py:791, ConversationHandler не tracking каждое сообщение. Чинить добавлением per_message=True.
- **PTBUserWarning JobQueue** — bot.py:819, обращение к app.job_queue без установленного [job-queue] extra. Фикс: добавить python-telegram-bot[job-queue] в requirements.
- **Приоритет**: низкий, не блокирует.
