## 2026-04-25 — Uniform-cut fallback алгоритм A

**Решение**: при невалидном Claude-selection резать видео на N равных кусков по cut_min_val минут с начала, последний усечённый.

**Альтернативы**:
- B: 2 куска по cut_min_val из центра видео
- C: 1 кусок cut_min_val с начала

**Почему A**: предсказуемость + полное покрытие видео. Пользователь явно выбрал длину нарезки → ожидает блоки этой длины, а не "случайную выборку". Покрывает начало и конец.

**Когда пересматривать**: после месяца в проде смотрим логи `[CUT] uniform-cut triggered`. Если процент срабатывания > 30% — алгоритм слишком часто активируется, может быть стоит сменить на B (фокус на центре). Если негативный фидбэк от пользователей — пробовать B или C.

## 2026-04-25 — UX uniform-cut: молча

**Решение**: без caption-пометки, без уведомления. Видео идёт как обычно.

**Почему**: пользователь не должен задаваться вопросом "что значит fallback". Снижает количество вопросов в саппорт.

## 2026-04-25 — Stable теги для якорей отката

**Решение**: после стабилизации сессии — тег `stable-intermediate-YYYY-MM-DD` на оба репо (api + bot).

**Почему тег, не ветка**: тег неподвижен, нельзя случайно перезаписать. Откат в одну команду.

## 2026-04-XX — MP4 H.264+AAC как единый внутренний формат

**Решение**: yt-dlp всегда конвертит в MP4 (H.264 + AAC).

**Почему**: максимальная совместимость для ffmpeg-нарезки (исключает кодек-конфликты) и для Telegram (плеер играет всё). Единый формат упрощает pipeline.
