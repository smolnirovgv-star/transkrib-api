Алгоритм:

### 4.2 — Оценка важности через Claude API
═══════════════════════════════════════════

Создай: backend/app/services/highlight_scorer.py

Промпт для Claude API:
"""
Ты анализируешь транскрипцию видео.
Оцени важность каждой фразы от 1 до 10.

Критерии высокой оценки (8-10):
- Ключевые идеи, выводы, инсайты
- Приветствие и прощание спикера → ВСЕГДА 9-10
- Конкретные факты, цифры, советы
- Эмоциональные моменты

Критерии низкой оценки (1-3):
- Слова-паразиты ("эм", "ну", "так сказать")
- Повторения
- Технические паузы

Верни только JSON массив без пояснений:
[{"index": 0, "score": 8, "reason": "ключевой вывод"}, ...]
"""

Группировка:
- Отбирать фразы score >= 6
- Объединять соседние в клипы 10-30 сек
- Если клип < 10 сек → объединить со следующим
- Если клип > 30 сек → разбить по внутренним паузам
- Приветствие/прощание: ВСЕГДА включать (первые и последние 30 сек)
- Итоговый ролик: минимум 20%, максимум 60% от оригинала

Обнови pipeline.py:
  Whisper → pause_detector → highlight_scorer → группировка → FFmpeg

### 4.6 — Транскрипт: просмотр и экспорт
═══════════════════════════════════════════

**A. Кнопка в ResultGallery с выпадающим меню**

В карточке результата рядом с кнопкой "Экспорт видео" добавь кнопку:
  📄 Транскрипт  ▾

Выпадающее меню (dropdown):
  👁  Смотреть в приложении   ← открывает TranscriptViewer
  ─────────────────────────
  ⬇  Скачать .TXT
  ⬇  Скачать .SRT
  ⬇  Скачать .JSON

Форматы файлов:
- TXT → чистый текст, каждая фраза с новой строки, без таймкодов
- SRT → стандартный субтитровый формат с таймкодами
- JSON → { segments: [{text, start, end, score}] } — для разработчиков

**B. Компонент TranscriptViewer.tsx**
Путь: platforms/desktop_windows/src/renderer/components/TranscriptViewer.tsx

Это отдельная страница/экран внутри приложения (не модал).
Открывается при выборе "Смотреть в приложении".

Структура страницы:
┌─────────────────────────────────────────────┐
│  ← Назад к результатам    📄 Транскрипт     │
│  [название видео / task_id]                 │
├─────────────────────────────────────────────┤
│  Фильтр: [Все фразы] [Только важные ≥6]    │
│  Поиск: [_________________________]         │
├─────────────────────────────────────────────┤
│  00:01:23  ████████░░  8/10                 │
│  "Сегодня мы разберём ключевые моменты..."  │
│                                             │
│  00:03:45  ██████░░░░  6/10                 │
│  "Первый важный принцип — это..."           │
│                                             │
│  (каждая фраза — карточка с таймкодом,      │
│   цветовой полосой важности, текстом)       │
├─────────────────────────────────────────────┤
│  [Скачать TXT] [Скачать SRT] [Скачать JSON] │
└─────────────────────────────────────────────┘

Детали UI:
- Полоса важности: зелёная (8-10), жёлтая (5-7), серая (1-4)
- Таймкод кликабелен → если есть плеер, перематывает на нужное место
- Поиск работает в реальном времени (фильтрует фразы)
- Кнопка "Назад" → возвращает на ResultGallery

**C. Backend endpoint для транскрипта**

GET /transcript/{task_id}?format=json
Возвращает: { segments: [{text, start, end, score, pause_before}] }

GET /transcript/{task_id}/download?format=txt|srt|json
Возвращает: файл соответствующего формата для скачивания

Транскрипт берётся из кэша Whisper (pipeline уже сохраняет его в storage).

**D. IPC handler (если Electron)**
