## ЗАДАЧА — Фаза 3.4: Автогенерация превью через Stability AI

### Что нужно сделать:

**1. Сначала прочитай эти файлы для контекста:**
- platforms/desktop_windows/src/renderer/components/ResultGallery.tsx
- platforms/desktop_windows/src/renderer/styles/globals.css
- platforms/desktop_windows/src/renderer/i18n/ru.ts
- backend/app/routers/ (посмотри что там есть)
- backend/app/config.py

**2. Добавь Stability AI ключ в конфиг**
В backend/.env добавь строку:
APP_STABILITY_API_KEY=your-key-here

В backend/app/config.py добавь поле:
stability_api_key: str = ""

**3. Создай сервис генерации превью**
Файл: backend/app/services/preview_service.py

Логика:
- Принимает: task_id (для получения транскрипции), или text (описание видео)
- Берёт первые 200 символов транскрипции как prompt для Stability AI
- POST https://api.stability.ai/v2beta/stable-image/generate/core
- Headers: Authorization: Bearer {key}, Accept: image/*
- Body (multipart): prompt, output_format=jpeg, aspect_ratio=16:9
- Сохраняет превью в %APPDATA%\Transkrib\storage\results\{task_id}_preview.jpg
- Возвращает путь к файлу

Обработка ошибок:
- Если ключ не задан → возвращает None (превью не генерируется)
- Если API недоступен → логирует ошибку, возвращает None
- Таймаут 30 секунд

**4. Добавь endpoint в backend**
В backend/app/routers/ добавь или обнови роутер:
POST /preview/generate
Body: { task_id: str }
Response: { preview_url: str | null }

Логика:
- Загрузить транскрипцию задачи из storage
- Вызвать preview_service.generate(text)
- Вернуть путь к превью

**5. Обнови ResultGallery.tsx**
- После завершения обработки — автоматически запрашивать превью (POST /preview/generate)
- Показывать превью как фоновое изображение карточки результата
- Если превью нет — показывать заглушку (текущее поведение)
- Добавить кнопку "Обновить превью" (иконка refresh)
- Показывать spinner пока превью генерируется

**6. Добавь i18n ключи в ru.ts, en.ts, zh.ts:**
